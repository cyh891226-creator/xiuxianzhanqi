# Figma提示词 - 任务与PVP系统

## 模块概述
任务系统是游戏的核心引导模块，包含任务列表、任务详情、任务追踪等功能。PVP系统包含竞技场、论道大会、门派战等玩法。

---

### 提示词

```
创建一个修仙题材回合制战棋游戏的任务中心界面。

布局要求：
1. 顶部导航栏（高度56px）：
   - 左侧：返回按钮
   - 中央：标题"任务中心"
   - 右侧：任务追踪按钮

2. 任务分类标签（高度48px）：
   - 全部 | 主线 | 支线 | 日常 | 周常 | 活动
   - 当前选中高亮显示

3. 任务列表（可滚动区域）：
   每条任务卡片样式：
   - 任务图标（任务类型）
   - 任务名称 + 星级难度
   - 任务描述（2行截断）
   - 进度条（如果有多阶段）
   - 奖励预览（图标+数值）
   - 操作按钮（前往/追踪/领取）

4. 主线任务高亮：
   - 金色边框
   - 置顶显示

5. 任务状态标识：
   - 进行中：蓝色标签
   - 已完成：绿色勾选
   - 可领取：金色脉冲
   - 已领取：灰色

设计风格：
- 任务卡片：白色圆角卡片
- 主线任务：金色边框
- 支线任务：白色边框
- 日常任务：蓝色边框
- 奖励：对应资源图标颜色
- 星级难度：1-5金色星星

交互效果：
- 完成任务有勾选动画
- 领取奖励有资源飞出
- 任务更新有闪烁提示
- 滑动切换有平滑过渡
```

## 2. 任务详情界面（Quest Detail Interface）

### 提示词

```
创建一个修仙战棋游戏的任务详情界面。

布局要求：
1. 任务头部：
   - 任务大图标（80x80）
   - 任务名称（粗体大字）
   - 任务类型标签
   - 星级难度

2. 任务描述：
   - 背景故事文本
   - 任务目标详解
   - NPC对话（如果有）

3. 任务目标列表：
   - 目标1：击败妖兽 x/10 ✓
   - 目标2：保护村民 x/5 [进行中]
   - 目标3：与村长对话 [未完成]

4. 奖励详情：
   - 经验奖励
   - 灵石奖励
   - 道具奖励
   - 声望奖励
   - 特殊奖励（装备/伙伴等）

5. 任务指引：
   - 目标位置提示
   - 地图跳转按钮

6. 操作按钮：
   - 前往执行
   - 放弃任务（红色，需要确认）

设计风格：
- 目标完成：绿色勾选
- 目标进行中：蓝色圆点
- 目标未完成：灰色圆点
- 奖励列表：图标+数值
- 放弃按钮：红色警告样式

交互效果：
- 点击目标有追踪动画
- 跳转地图有传送动画
- 放弃任务有二次确认
- 完成有金光效果
```

## 3. 任务追踪界面（Quest Tracking Interface）

### 提示词

```
创建一个修仙战棋游戏的任务追踪界面。

布局要求：
1. 追踪面板（可拖拽）：
   - 显示当前追踪的任务
   - 任务名称
   - 目标进度

2. 小型任务卡：
   - 任务图标（缩小版）
   - 任务名称（单行）
   - 进度条（精简）
   - 点击展开详情

3. 追踪数量：
   - 最多追踪3个任务
   - 超出提示

4. 地图标记：
   - 追踪的任务目标在地图上显示标记
   - 标记颜色根据任务类型

5. 快捷操作：
   - 一键追踪主线
   - 清除追踪

设计风格：
- 面板：半透明背景
- 进行中：蓝色高亮
- 标记点：对应类型颜色
- 追踪列表：简洁列表样式

交互效果：
- 拖拽有跟随效果
- 点击有展开动画
- 地图标记有呼吸动画
- 任务完成有消失动画
```

## 9. 排行榜界面（Ranking Interface）

```
创建一个修仙战棋游戏的排行榜界面。
```
